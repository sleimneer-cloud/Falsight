## 🚀 [FallSight] 팀원 온보딩: 깃허브 협업 & CI/CD 가이드

안녕하세요, FallSight(핑와) 팀원 여러분! 
우리는 13일이라는 짧고 타이트한 기간 내에 요양병원 낙상 감지 시스템을 완성해야 합니다. 
C++(카메라), Python(AI 추론), FastAPI(메인 서버), Qt C++(클라이언트)라는 4개의 독립된 시스템이 맞물려 돌아가기 때문에, **누군가의 코드 한 줄이 전체 시스템을 멈추게 할 수도 있습니다.**

이런 대참사를 막고 개발에만 온전히 집중하기 위해, 우리 프로젝트에는 **자동 검사 로봇(GitHub Actions)**과 엄격한 안전장치가 적용되어 있습니다. 

처음엔 낯설 수 있지만, 아래 가이드만 따라오시면 절대 코드를 날려 먹거나 시스템을 망가뜨릴 일이 없으니 안심하고 작업해 주세요!

### ❓ 깃허브 액션(CI)이 뭔가요? 왜 쓰나요?
쉽게 말해 `develop`(우리의 메인 코드) 성문을 지키는 **'로봇 수문장'**입니다.
여러분이 코드를 올리면, 이 로봇이 가상 환경에서 파이썬 문법이나 C++/Qt 컴파일 에러가 없는지 1차로 싹 검사해 줍니다.
* **장점 1:** 에러가 있는 코드는 로봇이 알아서 컷! 해주므로 통합 과정의 버그를 미리 막아줍니다.
* **장점 2:** 로봇이 통과시켜 줬다면 내 코드는 최소한의 안전성이 보장된 것입니다. 두려워 말고 팍팍 코딩하세요!

---

### 🛠️ FallSight 핵심 작업 순서 (이것만 기억하세요!)

우리는 `develop` 브랜치에 직접 코드를 올릴 수 없도록 자물쇠가 채워져 있습니다. 반드시 아래 5단계를 거쳐주세요.

**Step 1: 최신 코드 받고 내 작업실(Feature 브랜치) 만들기**
작업 시작 전, 항상 최신 상태를 유지하고 나만의 브랜치를 파고 들어갑니다.
```bash
git checkout develop
git pull origin develop
git checkout -b feature/본인기능명  # 예: feature/camera-stream, feature/login-ui
```

**Step 2: 지정된 폴더에서 코딩하고 푸시하기**
우리는 4개의 폴더(`camera_control`, `ai_server`, `main_server`, `client_qt`)를 사용합니다. 각자 맡은 폴더 안에서 작업한 뒤 내 브랜치에 코드를 올립니다.
```bash
git add .
git commit -m "feat: 15초 녹화 기능 추가"
git push origin feature/본인기능명
```

**Step 3: 결재 서류 올리기 (Pull Request, PR)**
깃허브 웹사이트에 들어가면 노란색 `Compare & pull request` 버튼이 뜹니다.
* 도착지(base)가 `develop`인지 확인하고 PR을 생성해 주세요. ("이 코드 합쳐주세요~" 하고 서류를 올리는 겁니다.)

**Step 4: 로봇의 검문 기다리기 (1~2분 소요)**
PR을 올리면 화면 하단에 노란색 동그라미가 돌면서 로봇이 검사를 시작합니다.
* ❌ **빨간 엑스가 떴다면:** 코드에 에러가 있다는 뜻입니다! 내 PC에서 에러를 고치고 다시 `commit & push` 하세요. (PR은 그대로 둬도 로봇이 다시 검사합니다.)
* ✅ **초록색 체크가 떴다면:** 무사히 통과! 

**Step 5: 리드의 최종 병합 (Merge)**
로봇이 초록불을 띄우면, 팀장이 코드를 최종 확인하고 `develop` 브랜치로 합쳐줍니다(Merge). 내 코드가 합쳐졌다면 다시 Step 1로 돌아가 다음 작업을 시작하면 됩니다.


### 📦 [매우 중요] 새로운 라이브러리/패키지를 사용했을 때 필수 규칙

우리 깃허브의 자동 검사 로봇(CI)은 **'아무것도 안 깔려 있는 완전 깡통 컴퓨터'**입니다. 
여러분이 로컬 PC에서 새로운 라이브러리(예: `fastapi`, `opencv`, `numpy` 등)를 설치해서 코드를 짰다면, **반드시 로봇에게도 "이거 설치해 줘!"라고 알려줘야 합니다.** 안 그러면 로봇이 패키지를 못 찾아서 에러(빨간불 ❌)를 뱉고 병합을 거부합니다.

**🐍 Python 노드 (Node 2 AI, Node 3 메인 서버)**
새로운 pip 패키지를 설치했다면, 반드시 해당 폴더의 `requirements.txt` 파일에 패키지명과 버전을 적어주세요.
* 방법 1: `requirements.txt` 파일을 열고 직접 타이핑 (예: `fastapi==0.103.1`)
* 방법 2: 터미널에서 `pip freeze > requirements.txt` 실행 후 커밋에 포함하기

**📷 C++ / Qt 노드 (Node 1 카메라, Node 4 관제 클라이언트)**
외부 라이브러리나 새로운 apt 패키지(예: `libopencv-dev`)가 필요해졌다면, 본인 PC에만 설치하고 넘어가면 절대 안 됩니다.
* 코드를 올리기 전, 반드시 **백엔드 리드(희석)나 팀 내 슬랙 채널에 "이러이러한 라이브러리가 추가로 필요합니다"라고 공유**해 주세요.
* 리드가 CI 로봇의 설정 파일(`ci.yml`)에 해당 패키지 설치 명령어를 추가해 주어야 로봇이 정상적으로 빌드할 수 있습니다.

### 💡 주의사항
* 프로젝트 루트 경로에 파일을 함부로 만들지 마세요. 반드시 지정된 4개의 폴더 안에서만 작업해야 합니다.
* 모르는 에러가 나서 로봇이 계속 빨간불을 띄운다면 언제든 리드에게 도움을 요청하세요!
